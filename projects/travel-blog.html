STACK

React, Redux, NodeJS, MongoDBAtlas, Express, SASS, GoogleAPI, Cloudinary

DURATION

Personal Project

1 month

GOAL

As I continue my journey of limiting the things I own + becoming a minimalist, I wanted to find a way to display all my sentimental items (train tickets, museum tickets, photobooth pics etc) and show pictures from my travels.

I also wanted to learn more about NodeJS + Express + having storage in the cloud.

CHALLENGE

I had to re-do the relationship schema when I was 3/4 of the way done. I thought about just pushing through and ignoring certain parts of the responses from the backend, but decided to humbly reevaluate the relationships - what better way to learn, right?

User Story

+ User is able to view all travel articles

+ User is able to view single travel article

+ User is able to view sentimental item images for single article + drag them around

+ User is able to view all the pictures for that travel

+ User is able to toggle view the trip's description

+ User is able to access link of where the picture was taken (if available)

User Story - Admin

+ User is able to login and be authorized using Gmail

+ User is able to create a trip with multiple destinations

+ User is able to upload pictures for single trip along with where the picture was taken + the Google link

+ User is able to upload sentimenal item images for a trip

+ User is able to edit a trip

+ User is able to delete a trip
