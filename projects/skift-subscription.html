Skift Subscription Project

STACK

React, Redux, NodeJS, MongoDB, Auth0, Recurly, Hubspot, Recharts, Wordpress, PHP

DURATION

1 of 3 Member Project @Skift

6 month+

GOAL

Create client page + admin page to keep track of product purchases. Client page that handles both individual and enterprise product purchases and have a dashboard view of all active/inactive subscriptions. Admin page shows dashboard of all users with and without purchases. Recurly and Hubspot handles secure credit card payments, email communications with client, and provide user data for internal marketing purposes.

MY RESPONSIBILITY

I was mostly incharge of writing NodeJS, React/Redux for client + admin sides of this project, but this product also exposed me to email html setup, and interacting with different API's + libraries (Hubspot, Auth0, Recurly, Rechart, etc)

User Story - Client side

+ User is able to purchase individual subscription

+ User is able to purchase enterprise subscription

+ User is able to create account to exisitng exterprise subscription

+ User is able to edit personal information on dashboard

+ User is view all products they have access to (active + inactive)

User Story - Admin side

+ User is able to log in securely if they have admin access

+ User is able to view total user activity for logging in, purchasing, and signing up

+ User is able to add, edit, and soft-delete a client

+ User is able to add, edit, and soft-delete an enterprise

+ User is able to add, edit, and expire purchases

+ User is able to add, edit, and delete products
